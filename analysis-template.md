# 缠论分析输出模板

完整盘面分析按以下结构组织。单点问题（"这是不是背驰？"）直接给四要素判断，不必全套。

## 模板

```
## [标的] [周期] 缠论结构分析

**结论先行**：[一句话：当前处于什么级别的什么结构，核心分类判断是什么]

### 一、结构现状
- 当下操作级别：X（依据：…）
- 中枢：区间 [ZD, ZG]，已震荡 N 段，状态（延伸/新生/扩展）
- 当前段位置：第几段、方向、是否接近终结（特征序列状态：有无缺口、是否回补）

### 二、力度判断
- [级别] + [a/c 两段] + [依据] + [背驰/非背驰/盘整背驰]
- 结构推论：（如"非背驰 ⇒ 新低必然存在，当前低点非真正底部"）

### 三、走势分类与推演
后续演化路径：
- 路径 A：[条件] → [对应买卖点] → [确认信号] → [目标/关注位]
- 路径 B：[条件] → [对应买卖点] → [确认信号] → [目标/关注位]
反证核对（若有结构必然性判断）：
- 推翻条件 1（次级别二买引发小转大 + 大阳线）：当前满足？
- 推翻条件 2（九段盘整化解）：当前满足？

### 四、应对标准
- 若 [信号1] 出现 → [动作/关注点]
- 若 [信号2] 出现 → [动作/关注点]
- 失效条件：[什么走势出现则本分析框架需重置]
```

## 写作风格要求

- **分类语言，不是预测语言**："演化路径有二：要么…要么…"；"新低不知是这次来还是下次来，届时按标准应对即可"。
- **每个判断带级别**："30分钟向下段""4小时级别中枢""1分钟三买"。
- **结构必然 + 但书**：说"必然"必须同时给推翻条件（小转大 / 九段盘整）。
- **锚定图表证据**：点位、中枢范围、段的划分都要与图上可见结构对应；作者已画的折线/箱体优先校验后引用，分歧处明说。
- 技术精确、直接，不加空洞的免责套话；但涉及用户实际下单决策时，一句话提醒：本分析是结构分类框架，具体仓位决策需用户结合自身情况判断。
- 可少量使用缠论体系的原文式表述增强精确性（"走势必完美""区间套""小转大"），首次出现且语境需要时用半句白话点明含义。

## 示例（浓缩版，展示语言风格）

> **结论先行**：BTC 4小时级别中枢构建中，本轮下跌段非背驰，结构上日线新低必然存在——当前低点不是真正底部；反弹按第三类卖点应对。
>
> 30分钟向下段回踩目标看至 85000 附近（对应前次级别中枢下沿），ETH 对应 2860 附近。反证核对：次级别未出现二买引发的小转大（无大阳线强拉），中枢延伸未达九段——两条推翻条件均不满足，新低预期维持。至于新低是本轮实现还是下一轮实现，不必追究也无须追究，三卖出现即是信号。
